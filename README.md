# 🔥주간알고🔥

서로의 성장을 돕는 SSAFY 14기 알고리즘 스터디 저장소입니다 💀

<br/>

## 💻 깃허브 이용 규칙

### 1. 브랜치 전략
매주 브랜치를 생성하지 않고, **본인의 이름으로 된 고정 브랜치** 하나를 계속 사용합니다.

1. **최초 1회**: 본인 이름의 브랜치 생성 (`git checkout -b 본인이름`)
2. **매주 작업 전**: `main` 브랜치의 최신 내용을 내 브랜치로 가져오기 (**필수!**)
   - `git pull origin main` (내 브랜치인 상태에서 입력)
3. **작업 후**: 내 브랜치에 커밋(`commit`) 하고 푸시(`push`)
4. **제출**: `main` 브랜치로 **Pull Request(PR)** 생성

### 2. 폴더 구조
브랜치는 하나지만, 파일이 섞이지 않게 **날짜(주차) 폴더**를 잘 만들어주세요.
```text
Algorithm-Study
├── 2026-01-07           # 해당 주차 날짜
│   ├── Gildong          # 본인 이름
│   │   ├── BOJ_1000.java
│   │   └── README.md
│   └── Cheolsu
│       └── ...
├── 2026-01-14
│   └── ...
└── README.md
```

### 3. 커밋 컨벤션 (Commit Message)
가독성을 위해 아래 형식을 준수해 주세요.
- `[플랫폼] 문제번호_문제이름 / 난이도 / 이름`
- 예시:
  - `[BOJ] 2557_Hello World / Bronze5 / 홍길동`
  - `[PGS] 42862_체육복 / Lv1 / 김싸피`

### 4. PR 규칙 (Pull Request)
- **PR 제목**: `[날짜] 이름 과제 제출`
  - 예시: `[260107] 홍길동 과제 제출`
- **내용**:
  - 이번 주 풀이한 문제들의 특이사항을 적어주세요.
- **Review**:
  - 스터디 모임 전까지, 다른 팀원의 코드에 **최소 1개 이상의 코멘트(칭찬, 질문, 개선점)** 를 남겨주세요.
- **Merge**:
  - 스터디 모임 종료 후 `main` 브랜치로 Merge 합니다.
